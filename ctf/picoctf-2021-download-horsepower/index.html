ctf
browser, chromium, v8

When the length of an array is set to a larger value than the number of elements by a bug, we call it an OOB array. In this post, I will solve a CTF challenge, introducing an exploitation technique that leverages an OOB array to implement limited exploitation primitives within the V8 sandbox and eventually achieve arbitrary code execution.

Bug

patch defines a new function SetLength() in JSArray class. It sets the length of the array to l without reallocation the array of elements.

patch also defines a new function ArraySetHorsepower(). It takes horsepower as an argument, and sets the length of the receiver to horsepower.

ArraySetHorsepower() is called to handle Array.prototype.setHorsepower() in JavaScript. We can set the length of an array to a large value to generate an OOB array.

We can access beyond the memory of the array to read or write the data of other objects.

Exploitation

Implementing addrof primitive

If an object is stored in an array, the array holds the address of the object in its memory.

We can place the OOB array and the array of objects consecutively by defining them in JavaScript one after the other as follows.

We can read the address of the object stored in obj_arr via oob_arr as a floating-point number. Then, we obtain the addrof() primitive, which returns the address of an arbitrary JavaScript object.

Implementing arbitrary address read and write

TypedArray in JavaScript is managed by JSTypedArray class.

There are two pointers in JSTypedArray. external_pointer is the address of the external array object that holds the elements of the TypedArray, and base_pointer indicates the distance between external_pointer and the actual address where the first element of the TypedArray is stored. In other words, the sum of external_pointer and base_pointer results in the address of the first element.

We can place the OOB array and the TypedArray consecutively by defining them one after the other, as we did when implementing addrof.

We can overwrite base_pointer and external_pointer of typed_arr with arbitrary addresses via oob_arr. Then, we can access the address to read or write via typed_arr.
