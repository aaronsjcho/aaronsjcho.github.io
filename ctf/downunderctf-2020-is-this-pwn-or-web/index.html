Setup

Run setup.zsh in the working directory.

patch.diff

Array.prototype.slice() is handled by ArrayPrototypeSlice().

HandleFastSlice() seems to be called when a fast path is available.

HandleFastSlice() modifies HandleFastSlice() a little. Instead of returning the array created by ExtractFastJSArray(), it sets the length of the array to count - start + SmiConstant(2) using ChangeLength().

ChangeLength() is a new function defined by patch.diff. It simply changes the length of the caller array to newLength, without reallocating the elements array.

Bug

count - start + SmiConstant(2) can be a negative integer, because count and start are the values taken by Array.prototype.slice() as its arguments. If so, an integer overflow occurs, and the new length of the sliced array is set to a large value.

When we access an element in the array, we can see the bounds-check routine as follows by setting a read watchpoint on the length field of the array.

At this point, rdx holds the address of the array, and r9 holds the index we’re trying to access. Builtins_KeyedLoadIC_Megamorphic() gets the length of the array from rdx + 0xb and compares it with the index. jae is an instruction that performs the unsigned comparison, which means that 0xfffffffe in the length field is treated as an unsigned integer. As a result, we can access an extensive index to read or write out of bounds of the array.

Exploitation

Implementing addrof primitive

We can place the OOB array and an array of objects consecutively by defining them in a row.

Then we can read the address of the object stored in obj_arr via oob_arr. Therefore, we can get the address of an arbitrary object.

Implementing arbitrary address read and write

TypedArray in JavaScript is managed by JSTypedArray class.

external_pointer is a compressed pointer that points to a ByteArray that holds the actual values stored in the TypedArray, and base_pointer is the distance between the external_pointer and the data start address.

We can place the OOB array and a TypedArray consecutively by defining them in a row.

After we overwrite the length of oob_arr, we can overwrite the base_pointer and external_pointer to arbitrary addresses via oob_arr. Therefore, we can manipulate the address that typed_arr accesses to read from or write to.
